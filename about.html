About

Andrew Blakey is a software developer who has been working in the industrial mobile robotics industry since 2013. Before that he completed a Bachelor's and Master's in Geography at the University of Waterloo.

He is a:

  • Roboticist-Geographer
  • Experienced team lead
  • Wearer of many hats

He works with:

  • ROS - the Robot Operating System
  • Python
  • TypeScript / JavaScript
  • C#
  • QGIS, ESRI Arc-All-The-Things
  • ENVI, ERDAS, Geomatica

He's led projects on all parts of the stack:

  • Web / mobile applications
    • UI/UX design, application architecture, implementation, testing, deployment
    • Robot mapping, configuration, task scheduling, monitoring
    • Fleet management and monitoring
  • Web servers and databases
    • Cloud, local server, on-robot
    • Real-time message-driven services and database APIs
    • Robot, site mapping, land/water surveying
  • Real-time robot fleet management
    • Centralized and peer-to-peer
    • Task allocation, queuing, docking, scheduling, safety behaviors
  • Industrial interfaces
    • Robot -> PLC communications
    • Fleet Management -> Modbus and OPC-UA
  • Robot communications
    • ROS, HTTP, WebSocket, WebRTC, Sockets (eg. ZeroMQ)
  • Data processing
    • Robotic water surveying
    • Indoor mobile robot mapping performance
